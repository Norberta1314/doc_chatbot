# 注意事项 
本文档罗列出的在执行重启操作后才能生效的命令，需要经过中兴通讯技术支持人员评估确认，在业务量较小的安全时间段（如凌晨开始）里执行，以避免对用户业务造成影响。 
# 平台高危命令 
## CommonS_TMSP 
命令|可能引起的后果/操作风险|是否重启|规避措施
---|---|---|---
SET SCALEGLBBASECFG|本命令用于对自动弹性功能的机制和流程相关的参数进行配置。当本命令中的弹性模式（mode）设置为自动（automatic）时，可能会触发系统自动弹性扩容或缩容操作。|不需要重启。|无
SET ROSNGINTERFACEKEEPALIVE|本命令用于接口处理单元（如IPU、PFU、FWU等）向主用Rosng协议栈单元（如OMU）注册成功后，修改主用Rosng协议单元检测接口处理单元状态的保活消息参数，便于NF根据业务特点调整主用Rosng协议栈单元检测接口处理单元状态的灵敏度。本命令执行后，如果参数设置不当，可能导致对接口处理单元在线离线状态检测错误。比如，如果不需要主用Rosng协议栈尽快感知接口处理单元状态变化，可以将参数保活时间间隔（INTERVAL）和保活次数门限（THRESHOLD）修改为一个较大的值。|不需要重启。|无
SET PASSWD|本命令用于修改系统用户密码。本命令执行后，会修改所有虚机的操作系统的用户密码，包括admin用户和root用户。密码修改不当可能导致系统用户权限问题进而影响系统功能。|不需要重启。|无
ACTIVATE NFS|本命令用于激活指定的NFS实例。激活NFS成功后，该NFS实例变为已部署状态。本命令执行后，成功激活的NFS可以正常对外提供业务功能。|不需要重启。|无
ADD NFS|本命令用于增加一个指定类型和规格的NFS实例。本命令执行后，会增加一条该类型和规格的NFS实例的记录，该NFS实例仍是未部署状态。|不需要重启。|无
RESET NFS|本命令将对指定NFS实例进行重启（复位）。本命令执行后，将导致业务中断并且服务不可用。可能会导致其他正在执行的命令执行失败并严重超时，因此如果正在执行其他命令时，不要进行重启NFS操作。|重启NFS，实际上是重启该NFS下的全部SC实例，重启成功后服务会恢复正常。|无
DEL NFS|本命令用于删除指定的NFS实例。本命令执行后，会彻底删除该NFS实例的记录，即通过执行SHOW NFS命令将无法再查询到该NFS实例的信息。|不需要重启。|无
MODIFY NFS|本命令用于修改指定的NFS实例的规格。本命令将删除NFS实例并根据版本中的NFS描述文件重新部署NFS实例。该操作将导致业务中断并且服务不可用，并且需要确保有足够的资源。|不需要重启。|无
DEACTIVATE NFS|本命令用于去激活某个NFS实例。本命令执行后，导致该NFS实例无法正常工作或对外提供服务，并且会删除该NFS的模型注册及相关配置数据，不会删除该NFS实例的记录，该NFS实例会变为未部署状态，该NFS实例下的所有SC全部会进行弹性缩容，并且无法正常工作或对外提供服务，同时释放其占用的部署资源。|不需要重启。|无
RESET SC|本命令用于重启（复位）指定的某个或某些SC实例。本命令执行后，SC会重启，重启过程中该SC提供的功能不可用。如果当前重启的SC上有业务运行，会导致业务受损。|SC会重启。|无
SET SC|本命令用于设置指定类型的SC的亲和标签以及最小SC组数等。使用本命令设置SC的亲和标签会删除并重新部署SC实例，可能导致SC实例部署失败。|不需要重启。|无
SWP SC|本命令用于倒换指定的SC实例。倒换SC成功后，业务会从原来的主用SC实例迁移到原来的备用SC实例，原来的备用SC实例将转为主用进而接替原来的主用SC实例正常工作，原来的主用SC实例转为备用。|不需要重启。|无
BLOCK SC|本命令用于闭塞指定的SC实例。闭塞SC成功后，该SC实例的业务会迁移到其他SC实例，该SC实例将成为闭塞状态。|不需要重启。|无
UNBLOCK SC|本命令用于解闭塞指定的SC实例。解闭塞SC成功后，该SC实例将从闭塞状态转为正常工作状态。|不需要重启。|无
INC SC|本命令用于增加指定SC类型的指定SC组内的SC实例。本命令执行成功后，将导致SC实例的弹性扩容，请确保有足够的资源。|不需要重启。|无
DEC SC|本命令用于减少指定SC类型的指定SC组内的SC实例。该操作将导致SC实例的弹性缩容，请提前做好业务迁移，以免造成业务损失。|不需要重启。|无
ACTIVATE SCGROUP|本命令用于激活指定的SC组。该操作将导致SC组的弹性扩容，请确保有足够的资源。|不需要重启。|无
DEACTIVATE SCGROUP|本命令用于去激活指定的SC组。该操作将导致SC组的弹性缩容，请提前做好业务迁移，以免造成业务损失。|不需要重启。|无
RESET VM|本命令用于重启指定的虚机。本命令执行后，可能导致部署在该虚机上的业务中断。|指定的虚机会重启。|无
SHUTDOWN VM|本命令用于关闭指定的虚机。本命令执行后，可能导致部署在该虚机上的业务中断。|不需要重启。|无
BLOCK VM|本命令用于隔离指定的虚机。隔离虚机是指将虚机设置为不能部署VRU，当进行弹性缩容或者弹性扩容等操作时，被隔离的虚机不可用。本命令执行后，会同步删除VRU，VRU将不能部署到该虚机上，通常情况下不建议使用此命令。|不需要重启。|无
RESET VRU|本命令用于重启指定的VRU。本命令执行后，会重启VRU，可能会对当前的业务造成重大影响。|VRU会重启。|无
REDEPLOY VRU|本命令用于重新部署VRU。本功能用于对业务进行测试时使用，会对当前的业务造成重大影响。|VRU会重启。|无
DEL NF|本命令用于删除指定的网元类型。删除正常或在用的网元类型将导致该类型的业务网元不能正常工作或提供服务。|不需要重启。|无
SET NF|本命令用于设置已有网元类型的类型别名以及NFID Node等信息。本命令执行后，可能影响系统运行，导致系统功能异常。|不需要重启。|无
SET MTU|本命令用于设置各个网络平面对应的MTU值。MTU值配置不当会导致配置的网络平面不通，会对当前的业务造成重大影响。|需要通过RESET SYSTEM命令重启整局。|无
DEL OSPATCH|本命令用于删除已加载、未激活的镜像补丁。镜像补丁文件被删除后，若当前系统已升级，则不能回退被删除的镜像补丁文件，可能导致整系统升级后回退失败。|不需要重启。|无
UNINSTALL OSPATCH|本命令用于卸载虚机上的镜像补丁。使用本命令将影响虚机的正常运行，被操作的虚机有可能会重启，导致业务呼损。请确保已经进行业务迁移。|不需要重启。|无
RESET SYSTEM|本命令用于重启（复位）整个VNF。本命令会重启整个VNF的所有虚机，会导致一段时间内系统无法正常运行。该过程耗时较长，请耐心等待系统重启。|整个VNF的所有虚机都会重启。|无
ACTIVATE PKG|本命令用于激活版本包。本命令执行后，系统版本会发生更新，可能影响系统运行，导致系统功能异常。|自动进行容器重建。|无
ACTIVATE HOTPATCH|本命令用于激活已经通过ADD HOTPATCH命令加载的热补丁包。若热补丁异常，本命令执行后会影响热补丁对应的业务功能。|不需要重启。|使用DEACTIVATE HOTPATCH命令去激活相应热补丁。
DEL HOTPATCH|本命令用于删除已去激活状态的热补丁包。热补丁包文件删除后，若当前系统已升级，且升级前已安装该补丁文件，则不能回退被删除的热补丁，可能导致整系统升级后回退失败。|不需要重启。|无
DEACTIVATE HOTPATCH|本命令用于去激活已经激活的热补丁包。本命令执行后，会影响热补丁对应的业务功能。|不需要重启。|无
ENABLE VERSION|本命令用于生效版本包。生效版本包就是使NF中容器运行数据库中已激活但容器并未运行的新版本生效，对应容器需要重建，影响版本对应NFS的业务功能。|自动进行容器重建。|无

DEL PLMN NI|该命令用于删除一个基于PLMN的网络标识配置。本命令配置成功后，对系统的影响如下：对于新接入用户，系统针对该PLMN将使用全局网络标识下发给UE，需要确认全局网络标识是否满足运营商要求。 对已经接入系统的用户，系统不会立即通知UE更新网络标识，用户下次接入时，才能再次触发系统下发网络标识。|不需要重启。|无

SET EMSPLUSLOG|该命令用于配置EMS PLUS上报日志的开关。本命令执行成功后，系统将根据重新控制上报EMS PLUS日志。启动EMS PLUS日志上报，对系统性能、网络带宽、对端EMS PLUS都会带来冲击，需要评估EMS PLUS日志量，预留足够资源，才可以开启。停止EMS PLUS日志上报，影响网络运维，需要确认关闭后满足运营商网络运维需求，才可以关闭。|不需要重启。|无

ADJUST BAKNE STATE|该命令用于强制调整备用网元状态。如果调整备用网元状态为强制备用网元退出服务（OUTSERVICE）：备用网元禁止业务处理，在检测到主用网元宕机的情况下，也不会处理业务。 如果调整备用网元状态为取消操作（CANCEL）：备用网元检测到主用网元宕机时，会进行业务处理。该操作只有在使用主备网元异地容灾部署组网时才需要执行，主备网元异地容灾部署时，POOL内两个异地的网元可以支持主备配置，其中主用、备用网元角色固定，备用网元仅在主用网元宕机时接替其工作，在主用网元恢复后退出服务。 该操作可以手动调整备用网元的工作状态，请谨慎使用。|不需要重启。|无
