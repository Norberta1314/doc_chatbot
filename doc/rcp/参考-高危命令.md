注意事项 : 
本文档罗列出的在执行重启操作后才能生效的命令，需要经过中兴通讯技术支持人员评估确认，在业务量较小的安全时间段（如凌晨开始）里执行，以避免对用户业务造成影响。

# 业务高危命令 

命令|可能引起的后果/操作风险|是否需要重启
---|---|---
DEL NFINFO|该命令执行后，可能会导致PCF向NRF注册的信息发生变化。已上线的用户后续业务流程失败或者数据无法存储到UDSF中。需要让已经上线的所有用户下线后再执行本命令。执行删除操作之后需要执行ADD NFINFO重新增加本局配置信息。|需要重启。执行RESET NFS命令重启Npcf_SMPolicyControl、Npcf_SIGLB服务或者使用RESET SYSTEM重启整个系统。
SET NFINFO|该命令执行后，可能会导致PCF向NRF注册的信息发生变化。已上线的用户后续业务流程失败或者数据无法存储到UDSF中。除了网元位置参数之外，修改其他参数前都需要让已经上线的所有用户下线后再进行修改。|需要重启。执行RESET NFS命令重启Npcf_SMPolicyControl、Npcf_SIGLB服务或者使用RESET SYSTEM重启整个系统。
SET PCFSERVICE|执行本命令之前之前请评估对系统的影响。例如，优先级改高可能导致大量用户接入本局，造成系统话务冲击。|不需要重启。
ADD SERVNUM|该命令执行后，PCF会向NRF注册此配置中的号段。增加通配的服务号段要谨慎，配置不慎可能导致业务都发送到本PCF，造成话务冲击。|不需要重启。
SET SERVNUM|该命令执行后，PCF会向NRF注册此配置中的号段。修改通配的服务号段要谨慎，配置不慎可能导致业务都发送到本PCF，造成话务冲击。|不需要重启。
SET USERKEYPRI|该命令执行后，可能导致已上线用户的后续业务流程失败。建议让所有用户下线后再执行本命令。|不需要重启。
CLEAR RULEDATA|该命令执行后，将删除策略管理所有配置数据，导致PCF无法下发策略给SMF。执行前请确认当前配置数据是否需要备份后再清空。|不需要重启。
SET SYSTEMSTATUS|该命令执行后，会导致系统所有用户重决策或者用户被强制下线。用户被强制下线会导致用户业务不连续。|不需要重启。
SET NFSTATUS|该命令执行后，会更新系统在NRF上注册状态，可能会影响系统正常处理业务。重复执行本命令的间隔建议超过10分钟，否则可能导致系统流程异常。|不需要重启。
ADD SCANCYC|该命令执行后，如果增加的配置不合理，可能导致系统会话数据被异常释放。|不需要重启。
SET SCANCYC|该命令执行后，如果配置不合理，可能导致系统会话数据异常释放。|不需要重启。
SET TIMER|该命令执行后，如果配置不合理，可能会导致本网元流程或者其他网元流程异常。如无特殊情况请保持默认值。|不需要重启。
SET SYS GLOBPARA|该命令执行后，如果配置不合理，可能导致系统某些功能异常。|不需要重启。
ADD DNNBSF|此配置用于指示哪些DNN的N7会话向BSF注册绑定信息。该命令执行后，如果配置错误可能导致PCF和NRF、BSF之间交互异常。|不需要重启。
SET DNNBSF|此配置用于指示哪些DNN的N7会话向BSF注册绑定信息。该命令执行后，如果配置错误可能导致PCF和NRF、BSF之间交互异常。|不需要重启。
SET MODULECAP|该命令执行后，如果容量改大，可能导致系统内存占用过大；如果容量改小，可能导致系统处理能力降低。修改短信数据区容量需要重启Npcf_SIGLB服务才能生效，修改业务事务数需要重启Npcf_SMPolicyControl服务才能生效。修改之前要评估对系统的影响。|需要重启。执行RESET NFS命令重启对应的服务。
